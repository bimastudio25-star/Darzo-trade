# Adelin v3 Composite Candidate Pack

This document describes the research-only candidate pack generated by the Adelin v3 composite detector.

Adelin v3 is not a live strategy. It does not send Telegram messages, call broker/order execution, modify Strategy 2, modify Strategy 3, modify VWAP, or modify market data.

## Why V3 Exists

Adelin v2 paused with a locked `INCONCLUSIVE` verdict. The v2 work showed that single-proxy candidates such as round levels and sweep extremes were not enough as a standalone detector.

Adelin v3 is a different hypothesis: require multiple independent conditions at the same time.

## Detector Scope

The foundation detector checks:

- C1: aged Daily or H1 swing liquidity level.
- C2: reaction zone beyond the swept level.
- C3: number theory confluence near the C1 level.
- C4: multi-timeframe liquidity confluence.
- C5: unambiguous reversal direction, with reaction zone beyond the swept level.

All checks use pre-anchor data only.

## How To Run

```bash
python scripts/create_adelin_v3_composite_candidate_pack.py --symbol XAUUSD --data-dir data --output-dir backtests/reports/adelin_v3_composite_candidate_pack --max-candidates 500 --dry-run
```

The script reads candle CSVs and writes a candidate pack. It does not run replay and does not produce matched controls.

## Output Files

- `candidate_pack.csv`: one row per v3 composite candidate.
- `generation_summary.json`: counts, rejection breakdown, safety flags, and replay gate verdict.
- `rejection_breakdown.csv`: why scanned windows were rejected.
- `decision_criteria.md`: locked future v3 replay criteria.
- `index.html`: pack overview.
- `examples/*.html`: candidate pages.
- `charts/*.svg`: M5 visual context charts.

## Replay Gate

If the candidate count is below 30, the pack verdict is `INSUFFICIENT_SAMPLE` and the next action is to pause v3. Do not relax detector conditions just to force sample size.

If candidates are at least 30, the next branch may run matched-control replay:

`feat/adelin-v3-candidate-pack-matched-control-replay`

That replay must use the pre-registered v3 criteria and must not optimize thresholds after seeing results.

## Interpretation

Candidate windows are not signals. This pack is a deterministic research sample for a new hypothesis. It makes no edge, PnL, deployment, or live-readiness claim.

## Initial Smoke Result

The first six-month local generation run used:

```bash
python scripts/create_adelin_v3_composite_candidate_pack.py --symbol XAUUSD --data-dir data --output-dir backtests/reports/adelin_v3_composite_candidate_pack --from-date 2025-11-19T00:00:00+00:00 --to-date 2026-05-19T23:59:00+00:00 --max-candidates 500 --dry-run
```

It produced 31 candidates, which clears the minimum `N >= 30` gate for the next matched-control replay branch. This does not validate the detector; it only means the sample-size gate is not immediately failed.
